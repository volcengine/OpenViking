# 轨迹：先建立证据表，再写周报

## 情境

输入材料来自发布记录、监控截图、会议纪要和成员口述，信息粒度不一致。
第一次直接按时间顺序整理，周报变成了活动流水账，读者无法快速判断结果。

## 成功执行

1. 先把事实归到“目标、结果、证据、影响、风险、下一步”六列。
2. 对每个结论寻找可验证证据；没有指标时使用明确的里程碑或验收结果。
3. 把同一目标下的多项活动合并成一个结果，不按人员或日期堆叠。
4. 先写一句话结论，再展开关键进展和偏差。
5. 无法确认的数字不补写，标记为“待确认”并列出缺少的证据。

## 结果

评审者能在首屏判断目标是否达成，且可以从每个结果追溯到事实。最终版本
从 1,900 字缩短到 1,150 字，没有损失关键决策信息。

## 可复用契约

- 周报写作从证据归类开始，而不是从润色开始。
- “完成了什么”必须和“产生了什么变化”放在一起。
- 对事实不足保持显式，不用合理猜测填满模板。
