# 轨迹：根据管理者反馈重写风险与计划

## 情境

初稿把风险写成“持续关注稳定性”“推进剩余迁移”，没有说明影响、负责人和
时间。管理者反馈这些表述无法支持决策，也无法在下个周期核验。

## 修正过程

1. 将风险改写为具体的不确定事件，而不是抽象主题。
2. 补充风险发生时会影响的范围或决策。
3. 每项缓解措施指定负责人和截止时间。
4. 把下周期计划改成可以验收的动作，包含交付物或退出条件。
5. 对延期项目单列“偏差与原因”，不把延期藏在进展描述中。

## 结果

评审者不再追问“谁来做、什么时候完成、做到什么程度算完成”。风险表可以
直接进入后续跟踪，下一期周报也有明确的回看基线。

## 可复用契约

- 风险至少包含事件、影响、措施、负责人和时间。
- 计划使用可观察的完成条件，避免只写“持续优化”“继续推进”。
- 延期说明事实原因和新决策，不写辩解性语言。
