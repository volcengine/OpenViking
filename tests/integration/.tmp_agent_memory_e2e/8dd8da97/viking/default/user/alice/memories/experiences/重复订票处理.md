## Situation
- 用户选定航班后调用订票工具，返回用户已存在相同日期的预订的失败提示

## Approach
- 告知用户存在重复预订的情况，说明已有的预订信息和新选择的航班信息，询问用户是取消原有预订换订新航班，还是两张都保留
- 若用户选择取消原有预订换订新航班，先调用cancel_booking工具取消原预订
- 验证原预订取消成功后，再次调用book_flight工具完成新航班的预订
- 预订成功后告知用户新票号，两张机票均生效
- 若用户选择保留两张预订，调用check_multi_booking_policy工具验证该日期是否允许多个预订
- 若政策允许，调用book_flight工具并传入override_duplicate=true参数完成新预订，告知用户两张机票均生效
- 若政策不允许，告知用户无法同时预订，需选择改订或取消原有预订
- 若用户选择仅保留现有预订，终止新预订流程并告知用户

## Reflect
- 不得在未征得用户同意的情况下擅自取消用户的原有预订
- 不得跳过取消原预订的步骤直接尝试重新预订新航班
- 不得混淆原有预订和新预订的信息，避免导致用户选择错误
- 不得在未告知用户冲突情况的前提下直接返回预订失败
- 不得在未验证原预订取消成功的情况下提交新的预订请求
- 不得未验证多预订政策就直接使用覆盖重复参数进行预订
