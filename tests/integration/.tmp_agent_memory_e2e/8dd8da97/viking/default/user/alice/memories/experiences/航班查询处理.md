## Situation
- 用户提出查询指定出发地、目的地、日期、时间段的航班需求

## Approach
- 确认用户提供的出发地、目的地、相对出行日期、时间段信息是否完整
- 将相对出行日期转换为对应的标准日期格式，确保与用户表述的相对日期一致
- 调用search_flights工具，传入出发地、目的地、标准日期、时间段参数
- 整理返回的候选航班信息，提供给用户选择

## Reflect
- 不得未经确认擅自修改用户提供的查询参数
- 不得在转换相对日期时出现错误，导致查询日期与用户需求不符
- 不得遗漏用户指定的时间段条件，返回不符合要求的航班
- 不适用于用户未提供完整查询必要信息的场景
- 不适用于机票预订、退改签等非查询操作场景
