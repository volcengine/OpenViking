## Design Context

### Users
- **主要用户**：AI/LLM 开发者，构建和调试 Agent 系统的工程师
- **使用场景**：个人开发调试为主，同时支持团队协作管理 Agent 上下文和知识库
- **使用环境**：长时间沉浸式工作，开发工具类产品，桌面端为主
- **核心任务**：管理 Agent 上下文数据、浏览资源文件、监控会话和操作状态

### Brand Personality
- **三词定义**：专业而有探索感、精确而开放、克制但有力
- **情感目标**：给用户信心感和掌控感，同时传递"这是一个充满可能性的工具"的探索精神
- **调性平衡**：像高端开发工具一样冷静可靠，但不冷漠——带有 Viking 探险者的锐利和果敢

### Aesthetic Direction
- **视觉参考**：Linear / Vercel 风格——简洁、精致、开发者导向的现代感
- **反面参考**：避免过度装饰、避免通用 AI 产品的 cyan/purple 渐变美学、避免过于花哨的仪表盘风格
- **主题**：深色和浅色双主题同等重要，都需要精心设计
- **字体**：当前使用 Noto Sans Variable，后续可根据需要升级
- **图标**：Lucide React

### Color Constraints
- **硬性约束：保持现有 OKLch 色彩 token 和主色调不变**
- 当前主色：oklch(0.5 0.134 242.749) — 沉稳的蓝色系
- 当前中性色已带品牌色相倾斜（蓝灰方向）
- 破坏性操作色：oklch(0.577 0.245 27.325) — 暖橙红
- 可以在不改变主色调的前提下微调辅助色、状态色等细节
- 深色模式使用低饱和度中性色 + 高对比前景色的方案保持不变

### Design Principles
1. **精确优先**：每个像素、每个间距都有意图。不使用通用 padding，spacing 要有节奏变化
2. **克制的力量**：装饰元素要少而精，让功能本身成为设计。信息密度高但不拥挤
3. **探索者的工具**：界面要让用户觉得自己在使用一个强大的专业工具，而非被引导的消费品
4. **双主题一等公民**：深色和浅色都是正式的、精心打磨的体验，不是互相的降级版本
5. **尊重现有系统**：在已建立的 OKLch 色彩体系和 shadcn/ui 组件架构上演进，而非推翻重来
